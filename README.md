# 한국어 맞춤법 검사기 (로컬 Claude)

웹사이트의 글자를 쓰는 칸(검색창, 글쓰기 칸 등)에서 **한국어 맞춤법을 자동으로 고쳐주는** 크롬 확장 프로그램입니다.

여러분 컴퓨터에 설치된 **Claude Code**가 맞춤법을 검사하므로, 별도의 API 키나 토큰 과금이 없습니다. (본인 Claude 구독 로그인 세션을 그대로 사용)

## 📥 다운로드

**[▶ 최신 버전 내려받기 (ko-spellcheck-dist.zip)](https://github.com/carsy078-maker/korean-spellchecker/releases/latest/download/ko-spellcheck-dist.zip)**

압축을 풀고 아래 "설치하기" 안내를 따르세요. (Python 설치 불필요 — 실행 파일이 포함돼 있습니다.)

---

## 이게 어떻게 작동하나요?

여러분 컴퓨터에 깔린 "클로드(Claude)"라는 프로그램이 맞춤법을 검사해 줍니다. 그래서 인터넷으로 돈이 나가지 않습니다. 대신, 여러분 컴퓨터에 클로드가 깔려 있고 로그인돼 있어야 합니다.

```
[웹사이트 입력칸] → 확장 → 로컬 Claude Code CLI → 교정 결과 → 오버레이 표시 → 적용
```

---

## ★ 먼저 준비할 것 (이게 없으면 작동하지 않아요)

### 1) 클로드(Claude) 유료 구독 (Pro 또는 Max 요금제)

### 2) 컴퓨터에 "Claude Code" 설치하고 로그인하기

아래 순서대로 하면 됩니다. 5분이면 끝나요.

**[1] "PowerShell"이라는 프로그램을 엽니다.**
- 키보드의 윈도우 키(⊞)를 누릅니다.
- `powershell` 이라고 입력합니다.
- 목록에 나온 "Windows PowerShell"을 클릭해서 엽니다. (파란색 또는 검은색 글자 창이 뜹니다.)

**[2] 아래 한 줄을 그대로 복사해서, 그 창에 붙여넣고 엔터를 누릅니다.** (붙여넣기: 마우스 오른쪽 버튼 클릭 또는 `Ctrl+V`)

```powershell
irm https://claude.ai/install.ps1 | iex
```

설치가 자동으로 진행됩니다. 끝날 때까지 잠시 기다립니다.

> 만약 위 방법이 안 되면, 대신 아래 한 줄을 붙여넣어도 됩니다:
>
> ```powershell
> winget install Anthropic.ClaudeCode
> ```

**[3] 설치가 끝나면, 같은 창에 아래를 입력하고 엔터:**

```powershell
claude
```

- 인터넷 창(브라우저)이 자동으로 열립니다.
- 여러분의 클로드 계정(1번의 유료 구독 계정)으로 로그인합니다.
- "로그인 완료" 같은 화면이 나오면 끝입니다. 창을 닫아도 됩니다.
- (이 로그인은 한 번만 하면 계속 유지됩니다.)

## 설치하기 (3단계, 5분이면 끝나요)

### 【 1단계 】 `install.bat` 파일을 두 번 클릭

- 검은 창이 뜨고 **"설치 완료!"** 라는 글자가 나오면 성공입니다.
- 만약 파란 경고창("Windows의 PC 보호")이 뜨면, **"추가 정보" → "실행"** 을 눌러주세요. (안전한 파일입니다.)

### 【 2단계 】 크롬(Chrome)에 확장 프로그램 등록 (한 번만)

1. 크롬을 열고, 맨 위 주소창에 아래를 그대로 입력 후 엔터:
   ```
   chrome://extensions
   ```
2. 화면 오른쪽 위에 있는 **"개발자 모드"** 스위치를 켭니다. (파란색이 되면 켜진 것)
3. 왼쪽 위에 나타나는 **"압축해제된 확장 프로그램 로드"** 버튼을 클릭합니다.
4. 폴더 선택 창이 뜨면, 이 폴더 안의 **`extension`** 폴더를 선택하고 "폴더 선택"을 누릅니다.

→ 목록에 "한국어 맞춤법 검사기"가 생기면 등록 성공입니다!

### 【 3단계 】 툴바에 아이콘 고정하기 (버튼으로 쓰려면 필요해요)

확장을 등록해도 크롬 오른쪽 위 툴바에 아이콘이 바로 안 보일 수 있습니다. 아래처럼 고정하면 항상 보입니다.

1. 크롬 오른쪽 위의 **퍼즐 조각 모양 아이콘(🧩)** 을 클릭합니다.
2. 목록에서 "한국어 맞춤법 검사기" 를 찾습니다.
3. 그 옆의 **압정(📌) 모양 아이콘** 을 클릭합니다. (파란색이 되면 고정된 것)

→ 이제 툴바에 아이콘이 항상 보이고, 클릭해서 쓸 수 있습니다. (아이콘을 안 쓰고 `Ctrl+Shift+K` 단축키만 써도 됩니다.)

---

## 사용하는 법

1. 아무 웹사이트나 열고, 글자를 쓰는 칸을 한 번 클릭합니다. (예: 네이버 검색창, 메일 쓰기, 블로그 글쓰기 칸 등)
2. 키보드에서 **`Ctrl` + `Shift` + `K`** 를 동시에 누릅니다. (또는 크롬 오른쪽 위 확장 아이콘을 눌러 "현재 입력창 검사")
3. 화면 오른쪽 아래에 검사 결과가 나타납니다. 틀린 곳이 노란색으로 표시됩니다.
4. **"적용"** 버튼을 누르면 고쳐진 문장으로 바뀝니다. 마음에 안 들면 "취소"를 누르면 됩니다.

> ⚠️ 크롬 설정 화면이나 새 탭 화면에서는 작동하지 않습니다. 반드시 일반 웹사이트에서 사용하세요.

---

## 잘 안 될 때

**"이 페이지에서는 실행할 수 없습니다" 라고 뜰 때**
→ 크롬 설정 화면이나 새 탭에서 누른 것입니다. 네이버 같은 일반 웹사이트를 열고 다시 해보세요.

**결과가 안 나오거나 "연결" 관련 오류가 뜰 때**
→ `install.bat` 을 다시 두 번 클릭한 뒤, `chrome://extensions` 화면에서 이 확장의 새로고침(↻) 버튼을 누르세요.

**"로그인이 안 되어 있다(Not logged in)"는 오류가 뜰 때**
→ 검은 명령창에서 `claude` 를 실행해 로그인을 확인하세요.

## 컴퓨터를 껐다 켠 뒤에는? (다시 설치 안 해도 돼요)

설치(`install.bat`)는 **딱 한 번만** 하면 됩니다. 컴퓨터를 재시작해도 설정은 그대로 남아 있으니 다시 설치할 필요가 없습니다.

→ 재시작한 뒤에는 그냥 크롬을 열고 바로 사용하면 됩니다. (웹사이트 입력칸 클릭 → `Ctrl + Shift + K`)

- **참고 1:** 재시작 후 크롬을 처음 켜면 "개발자 모드 확장 프로그램을 사용 중지하시겠어요?" 같은 안내가 뜰 수 있습니다. 이때 "사용 중지"를 누르지 말고, 그냥 **X(닫기)** 를 누르세요. 그러면 계속 켜진 채로 유지됩니다.
- **참고 2:** 혹시 맞춤법 검사가 안 될 때는 `install.bat` 을 다시 하기 전에, 먼저 `chrome://extensions` 화면에서 이 확장의 새로고침(↻) 버튼을 한 번 눌러보세요. 대부분 이걸로 해결됩니다.
- **`install.bat` 을 "다시" 실행해야 하는 경우는 딱 하나입니다:** 이 폴더를 다른 위치로 옮기거나 폴더 이름을 바꿨을 때. (그럴 땐 옮긴 뒤 `install.bat` 을 다시 두 번 클릭하세요.)

## 삭제하려면

- 이 폴더의 `uninstall.bat` 을 두 번 클릭하세요.
- 그리고 `chrome://extensions` 에서 이 확장을 "삭제"하면 됩니다.

## 개발자용 메모

- **구조:** 크롬 확장(Manifest V3) ↔ 네이티브 메시징 ↔ 파이썬 호스트(`host/host.py`)가 `claude -p ... --output-format json` 실행 → 교정 JSON 반환.
- **인증:** 호스트가 매 호출마다 `ANTHROPIC_API_KEY` 환경변수를 제거해 **구독(OAuth) 세션**으로 동작합니다. (`--bare` 는 OAuth 세션을 읽지 않으므로 사용하지 않고, `--system-prompt` + `--tools ""` 사용)
- **빌드:** `host.py` → `host.exe` 는 PyInstaller(`--onefile`)로 컴파일. 배포 zip 은 `package.ps1` 로 생성(비공개 키 `key.pem` 제외).
- **고정 확장 ID:** `extension/manifest.json` 의 `key` 필드(공개 키)로 어느 PC에서든 ID가 `igigjjnjaoalnlgnbcpbcbojihfdoleh` 로 고정됩니다. 서명용 비공개 키 `key.pem` 은 저장소에 포함되지 않습니다.
- **한계:** 로컬 Claude Code CLI + 로그인 + 본인 구독이 각 PC에 있어야 합니다. 불특정 다수 배포에는 API/서버 방식으로의 전환이 필요합니다.
